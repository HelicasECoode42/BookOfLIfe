# Demo Walkthrough

这份文档用于现场演示，不是开发设计文档。

## 演示前准备

1. 在项目根目录配置 `.env`
2. 启动后端：

```bash
npm start
```

3. 打开前端首页：

- `index.html`

4. 演示前跑一次检查：

```bash
npm run stage2:smoke
npm run stage9:backend-smoke
npm run stage11:photo-smoke
npm run stage12:avatar-smoke
```

## 演示顺序

### 第一段：先让观众知道它不是普通聊天页

展示：

- 首页封面
- 聊天入口
- 记忆页入口
- 图谱页入口

一句话说明：

> 这不是单纯的 AI 对话，而是把对话逐渐收进一本“人生之书”。

### 第二段：展示对话控制能力

建议输入一组带修订的测试：

```text
今天和朋友出去吃饭了
哦不对，那是昨天
不是朋友A，是朋友B
```

要强调：

- 系统不会继续沿用旧时间
- 系统不会继续沿用旧人物
- 不确定时会先确认

### 第三段：展示候选线索

继续输入一组相对完整的生活片段。

然后展示：

- 候选线索如何生成
- 不是直接强存，而是先进入待整理

### 第四段：展示草稿确认

点击“收进这一页”或“整理成草稿”。

展示：

- AI 提炼出的人物 / 时间 / 事件
- 用户可以确认保留，而不是被系统直接写死

### 第五段：展示图谱和人生线

打开 `memory-map.html`。

展示：

- 人物线
- 时间线
- 待整理线索
- 人生线摘要

一句话说明：

> 系统不是在堆聊天记录，而是在把离散片段逐渐结构化。

### 第六段：展示照片墙和数字人

先去设置页生成一次温伴形象。

然后展示：

- 首页和聊天页已经稳定显示新的形象
- 这不是前端临时图，而是后端落盘后回显

再去记忆图谱页上传一张照片。

展示：

- 照片会被收进照片墙
- 可以手动挂到人物或记忆
- 刷新后仍然能看到

### 第七段：展示后台增强面板

展示右下角的后台增强面板。

要强调：

- 主回复先返回
- 回复后整理、照片上传、形象生成都走慢链
- 所有增强任务都不会阻塞聊天主链

### 第八段：展示后端恢复能力

可选演示：

1. 刷新页面
2. 重新打开聊天页
3. 展示状态依然能恢复

要强调：

- 现在不再只是浏览器临时缓存
- 服务端会保留关键状态与 reply trace

## 演示时避免的坑

- 不要临场输入过长、过散、无明显结构的话
- 不要一开始就测非常复杂的长人生回忆
- 不要现场改代码后马上演示
- 演示前确认 3001 端口后端已经启动

## 演示时最值得强调的四点

1. AI 会先收口，不乱说
2. 记忆进入系统前会先经过整理和确认
3. 语音、照片墙、数字人都已经被收进同一条异步增强链
4. 现在已经有后端状态恢复，不只是一次性网页原型
